# Pyramid Thinking (Internal Reference)

> **Preamble:** The Pyramid Principle is a **structural thinking tool** for organizing arguments, assumptions, and evidence. In this skill, it is applied in Phase 3 to structure assumptions and build the logic tree. The same structure later becomes the deck argument in Phase 6, but the thinking happens here.

---

## 1. The Pyramid Principle

Based on Barbara Minto's framework. The core structure for logical, persuasive arguments.

**Core insight:** The clearest sequence is to order from the top down — summarize first, then provide details. Without a predefined grouping, different listeners impose different structures on the same information.

### Three Rules of Pyramid Structure
1. Ideas at any level must always be **summaries** of the ideas grouped below them
2. Ideas in each grouping must always be the **same kind of idea**
3. Ideas in each grouping must be **logically ordered** using one of four methods:
   - **Deductively** (major premise, minor premise, conclusion)
   - **Chronologically** (first, second, third)
   - **Structurally** (grouped by category or domain)
   - **Comparatively** (ordered by importance, most critical first)

## 2. Pyramid Substructures

Three substructures that reveal how ideas connect:

### Vertical Relationship
- Making a statement that tells the reader something they don't know automatically raises a logical question: "Why?" or "How?" or "Why do you say that?"
- The next level down must address that question
- This creates a dialogue: claim → question → support → deeper question → deeper support

### Horizontal Relationship
- Ideas at the same level should relate together either inductively or deductively
- Horizontal flow connects all key elements of the story and builds to the conclusion
- **Test:** Read ONLY the top-level ideas in sequence. Does the story flow? Does each lead naturally to the next?

### How This Applies to Assumptions
- Each assumption IS a claim that raises a question ("Why do we believe this?")
- The evidence supporting it IS the next level down
- Assumptions at the same level should relate horizontally — they should be the "same kind of idea" and follow a logical order
- The horizontal test on leadlines IS the Pyramid horizontal relationship test

## 3. Deduction vs. Induction

Two ways to structure the relationship between assumptions:

### Deductive Logic
1. Make a statement about a situation that exists in the world
2. Make another statement about a situation that exists at the same time
3. State the implication of these two situations co-existing

Good for causal chains: "A is true AND B is true, THEREFORE C follows." Each assumption builds on the previous one. The dependent assumption comes AFTER its prerequisite.

### Inductive Logic
1. Notice that several different things (ideas, events, facts) are similar in some way
2. Bring them together in a group
3. Comment on the significance of their similarity

Good for grouping independent evidence: "Here are three reasons this works." The "plural noun" names the group.

### Recognizing Poor Inductive Arguments
A poor inductive argument groups ideas but draws an inference that doesn't follow:

**Bad example:**
- Subpoints: "Don't face reality / Won't countenance internal criticism / Won't cut off losing activities"
- Inference: "Managers mismanage because they want to"
- **Problem:** The subpoints describe behaviors, but the inference attributes intent. The grouping doesn't support the conclusion.

Work from the bottom layer up — identify what CAN be inferred from the grouping, not what you wish were true.

## 4. Building a Pyramid (Top-Down)

### Step-by-step:
1. **Fill in the Top Box:**
   - What Subject are you discussing?
   - What Question are you answering?
   - What is the Answer? (the SCQA Answer from Phase 1-2)
2. **Match the Answer to the Introduction:**
   - Does the SCQA still hold?
   - Do the Question and Answer still follow from S and C?
3. **Find the Key Line:**
   - What New Question is raised by the Answer?
   - Will you answer it deductively or inductively?
   - If inductively, what is your plural noun? ("three reasons," "four factors")
4. **Structure the Support Points:**
   - Repeat the question/answer process at each level
   - Each leadline IS a level of the pyramid

### Bottom-Up Alternative
If top-down doesn't work (thinking not well-enough developed):
1. List all the points you want to make
2. Work out the relationships between them
3. Draw conclusions

## 5. Leadlines

- A leadline states the main analytical takeaway — not a label or topic
- Each leadline connects to the pyramid structure and tells a story horizontally across the argument
- **Leadline rules:**
  - Positive analytical claims, not testable hypotheses
  - No hedge language ("sufficient," "adequate," "may," "could")
  - No embedded statistics — stats belong in the supporting evidence, not the leadline
  - Each leadline is a complete sentence a decision-maker can act on
- **Horizontal test:** Reading ONLY the leadlines should tell the full recommendation story

---

## 6. Logic Tree Structure

The logic tree IS the pyramid applied to a business case:

```
Key Problem / Question (from SCQA)
├── Selected Solution / Answer (from Phase 2)
│   ├── Key Assumption 1 → Leadline 1
│   │   ├── Supporting Data / Logic
│   │   └── Supporting Data / Logic
│   ├── Key Assumption 2 → Leadline 2
│   │   └── Supporting Data / Logic
│   └── Key Assumption 3 → Leadline 3
│       └── Supporting Data / Logic
```

The four columns:
1. Key Problem / Question (SCQA Q)
2. Selected Solution (SCQA A)
3. Key Assumptions → Leadlines (structured by pyramid ordering)
4. Supporting Logic / Data for each assumption

### Evidence Audit Pattern

When enriching the logic tree with evidence (Phase 5, after research):
1. **Source mapping:** For each leadline, list every supporting source by ID from the Source Ledger
2. **Strength rating:** Rate evidence strength per leadline based on source count, verification status, and tier distribution
3. **Horizontal leadline test:** Read only leadlines in sequence — does the story flow?
4. **Sensitivity nodes:** Which variables, if changed, swing the entire recommendation?
5. **Flag weaknesses:** Identify leadlines with the thinnest evidence — these are where the argument is most vulnerable
