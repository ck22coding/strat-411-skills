# Phase 2: Solution Development — Detailed Steps

> Loaded by SKILL.md at the Phase 2 gate. Follow all steps in order. Use the section matching the case type (Platform or Standard).

---

## Platform Flow

If the case involves a platform business facing a growth/scope question (identified in Phase 1), follow these steps instead of the Standard Flow.

**Step 1a: Map Historical Growth Moves (2x2 Matrix)**
Before proposing anything new, map every growth move the platform has already attempted into the 2x2 growth matrix, with dates. This shows where the company has concentrated effort and which quadrants are underexplored. Present as an ASCII 2x2 grid with dated entries. Ask: "Does this match your read of the case? Anything missing?"

**Step 2a: Establish Unifying Theme**
Ask first: "What vision or theme should guide the platform's expansion? This determines what belongs in the portfolio and what doesn't." Share contrasting examples of broad vs. narrow themes from `references/platform-strategy.md`. Refine together.

**Step 3a: Map New Growth Opportunities (2x2 Matrix)**
Walk through each quadrant systematically, generating options consistent with the chosen theme. This is a MAP of possibilities, not alternatives to choose between. Overlay on the historical matrix so gaps are visible.

**Step 4a: Assign Governance Mode**
For each proposed interaction: own (build in-house), partner (equity/co-development), or affiliate (open APIs)? Use the value curve logic from `references/platform-strategy.md`.

**Step 5a: Sequence the Portfolio**
Order moves by: (a) leverage of existing assets, (b) cross-side synergy with core, (c) investment required. Present the proposed sequence and ask if the ordering makes sense.

**Step 6a: Select First Investment for Deep-Dive**
Ask the user to pick the first move for detailed analysis in Phases 3-6.

**Gate:** Portfolio mapped, theme confirmed, sequence roughed out, first investment selected. **Do NOT proceed to Phase 3 until confirmed.**

## Standard Flow

### Step 1: Generate Candidates

Ask first: "What solutions can solve the problem the firm is facing?" Restate the Complication and Question from Phase 1's SCQA framing before asking, so the user is grounded. Wait for their response.

Then generate 3-5 candidate solutions answering: "How should the firm overcome this core problem?" — exhausting the solution space before committing prevents anchoring on the first good idea. Include the user's suggestions refined as needed, and add distinct options to reach 3-5 total.

When generating candidates, specifically consider:
- The firm's current positioning and how to leverage it
- Resources and capabilities that could be redeployed
- The direction the industry appears to be heading
- Think across: growth, cost, competitive response, repositioning, capability building

**Apply theory lenses.** Read `references/dynamic-capabilities.md` and `references/competitive-positioning.md`. Would a Schumpeterian creative-destruction move work? Does the rugged landscape suggest a long jump vs. hill-climbing? Can existing routines support the change? Where does competitive positioning create white space?

### Step 2: Challenge the Configuration

After presenting candidates, ALWAYS ask: "Before we refine — are we optimizing the current approach, or should we consider a fundamentally different configuration? Is there a version of this where the firm does something radically different rather than improving what it already does?"

If the student expresses interest in exploring radical alternatives, offer: "Would you like to map the industry's competing factors to see where white space exists? We can build a quick canvas showing what the industry competes on and where a different approach might break the pattern."

If they accept, walk them through building a strategy canvas: identify 5-8 competing factors the industry competes on, rate each player, then use the Eliminate/Reduce/Raise/Create framework to design a differentiated configuration. Read `references/competitive-positioning.md` for the framework details.

### Step 3: Refine

Present the full candidate set (2-3 sentences each). Ask:
- Any solutions to modify, combine, or remove?
- Any to add?

**Do NOT proceed until the candidate set is finalized.**

### Step 4: Select

Use `AskUserQuestion` (multi-select) to ask which candidates to explore further.

Frame the output: "Our Answer to the Question '[Q from Phase 1]' is [selected solution]." This completes the SCQA — the selected solution IS the Answer.

**Do NOT proceed to Phase 3 until a selection is made.**
