# Output Convention

> Template asset — read at Phase 1 to establish file naming.

## Case Prefix

Ask: "What short name should I use for file naming? Examples: `airbnb-case`, `dell-case`, `spotify-case`." Use lowercase, hyphenated. This prefix applies to all output files.

## Working Directory

Ask: "Where should I save working files? I recommend creating a folder on your Desktop — e.g., `~/Desktop/{case-prefix}/`." If the user doesn't specify, default to `~/Desktop/{case-prefix}/`.

Create the directory if it doesn't exist.

## Naming Convention

All files follow: `{working-directory}/{case-prefix}-{descriptor}.{ext}`

**Save instruction rule:** At every phase gate that produces a file, explicitly state the save path using this convention.
